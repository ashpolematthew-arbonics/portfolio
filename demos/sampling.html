📊 Stratification & plot sampling statistics

Field inventory is the most expensive part of an MRV programme, so you sample as few plots as you can while still meeting the methodology's precision target. This tool sizes a stratified field campaign: enter each stratum's area and variability, set your precision target, and it computes the VCS-compliant plot count, the optimal allocation across strata, and the uncertainty deduction you'd incur.

How this works — the method behind the demo

For a target relative error E at confidence t, the stratified sample size with Neyman optimal allocation (large-population form) is:

n = (t / E)² · (Σ Wh·CVh)²

where W_h is each stratum's area weight and CV_h its coefficient of variation. Plots are allocated nh = n · (WhCVh) / Σ WhCVh — more plots go to bigger, more variable strata. The naïve simple-random alternative uses the pooled CV √(Σ WhCVh²); the difference is the efficiency you buy with stratification.

The uncertainty deduction follows the VCS rule of thumb: when the 90% confidence half-width exceeds 10% of the estimate, the creditable amount is reduced by the excess — a direct financial incentive to sample enough.

How this works — the method behind the demo

Each pixel carries a spectral feature vector (here two standardised bands — think NDVI and a SWIR/NIR index). k-means partitions the feature space into k compact clusters by iteratively assigning pixels to the nearest centroid and recomputing centroids until convergence.

A latent biomass value is correlated with the spectral features, so spectrally-homogeneous clusters are also biomass-homogeneous. Variance explained is R² = 1 − (Σ Wk·σ²k) / σ²total — the share of biomass variance the strata capture. Because required plots scale with CV², cutting the pooled within-stratum CV translates directly into fewer plots (shown above).

X-Means automates the choice of k: it fits k-means for a range of k and selects the value that minimises the Bayesian Information Criterion (BIC), balancing fit against model complexity — the same family as Weka's XMeans. The lower panel plots BIC vs k with the selected value marked.
