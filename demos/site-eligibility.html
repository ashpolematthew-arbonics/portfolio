🛰️ Drone site eligibility assessment

Not every hectare can host a carbon project. Within a single project boundary, eligibility is decided pixel by pixel against a stack of rules, then the passing pixels are dissolved into a clean vector eligibility layer. Toggle each rule and watch the raster re-classify in real time; switch to the vectorised view to see the eligible area as polygons — the form it takes when it enters a project's GIS.

How this works — the method behind the demo

Eligibility screening is a raster overlay. Inside the project boundary, every pixel is sampled from co-registered geospatial layers: a land-cover time series (Sentinel-2 / Dynamic World for the 10-year forest test), a DEM-derived slope raster, peat & wetland maps, protected-area boundaries (WDPA), and a prior land-use classification — all modelled here as spatially-coherent synthetic fields.

A pixel is eligible only if it passes every enabled rule — a logical AND across the criteria rasters. Disabling a rule relaxes the screen, so the eligible area grows. Eligible area × a nominal 8 tCO₂e/ha/yr gives a first-order credit estimate.

The vectorised view dissolves contiguous eligible pixels and traces their boundary edges — the raster-to-vector step that turns a classification grid into the polygon eligibility layer a project actually files. In production this runs at 10 m Sentinel resolution (or finer from drone orthomosaics) across the whole boundary.
